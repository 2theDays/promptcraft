# 🚀 범용 AI 프롬프트 빌더 — 바이브 코딩 개발 로드맵

> 비개발자가 AI의 도움으로 실제 서비스를 만드는 완전한 설계 가이드

---

## 📌 앱 개요

**목표**: 누구나 ChatGPT, Claude, Gemini 등 다양한 AI에 사용할 수 있는 고품질 프롬프트를 쉽게 만들 수 있는 웹 서비스

**핵심 가치 제안**: "AI를 잘 쓰고 싶은데 프롬프트를 어떻게 써야 할지 모르겠다" → 이 앱이 해결해 준다

---

## 🎯 핵심 기능 설계 (MVP)

### 반드시 있어야 할 기능 (1단계)

1. **프롬프트 생성기** — 목적, 톤, 대상, 출력형식을 입력하면 프롬프트 자동 생성
2. **카테고리별 템플릿** — 마케팅, 글쓰기, 코딩, 번역, 요약 등 분야별 프롬프트 모음
3. **복사 기능** — 생성된 프롬프트를 클립보드에 한 번에 복사
4. **저장/즐겨찾기** — 마음에 든 프롬프트 저장

### 2단계에서 추가할 기능

5. **프롬프트 히스토리** — 이전에 만든 프롬프트 조회
6. **평점/공유** — 커뮤니티에 프롬프트 공유 및 평가
7. **AI 실시간 테스트** — 만든 프롬프트를 앱 안에서 바로 테스트
8. **사용자 계정** — 로그인 후 개인 프롬프트 라이브러리 관리

---

## 🛠️ 기술 스택 (비개발자용 최적 선택)

| 역할 | 도구 | 이유 |
|------|------|------|
| **화면(UI) 제작** | React + Tailwind CSS | AI가 코드를 99% 작성해줌 |
| **백엔드/DB** | Supabase | 코드 없이 데이터베이스·로그인 설정 가능 |
| **AI 연동** | Claude API 또는 OpenAI API | 프롬프트 생성에 AI 활용 |
| **배포** | Vercel | GitHub 연동 후 버튼 하나로 배포 |
| **바이브 코딩 도구** | Claude (Cowork) + Cursor IDE | 한국어로 원하는 것 설명하면 코드 생성 |
| **버전 관리** | GitHub | 코드 저장소, 무료 |

**총 예상 초기 비용**: 월 0~20달러 (트래픽이 적을 때는 무료)

## 📅 단계별 개발 계획

### 1단계: 기획 및 설계 (1~2일)

**목표**: 앱의 구조를 명확히 정의

**할 일**:
- [ ] 타겟 사용자 1명을 구체적으로 떠올리기 (예: "AI를 업무에 쓰고 싶은 30대 직장인")
- [ ] 핵심 기능 3개만 정하기 (욕심 부리지 말 것)
- [ ] 화면 구성 손으로 스케치 (메모지에 박스 그리기만 해도 됨)
- [ ] 앱 이름, 색상 컨셉 결정

**바이브 코딩 프롬프트 예시**:
> "타겟 사용자가 AI 활용하고 싶은 직장인인 범용 프롬프트 빌더 앱의 주요 화면 5개를 설계해줘. 각 화면에 필요한 요소들을 설명해줘."

---

### 2단계: 환경 세팅 (반나절~1일)

**목표**: 개발 도구 설치 및 프로젝트 초기화

**순서**:

1. **Node.js 설치** → nodejs.org에서 LTS 버전 다운로드
2. **Cursor IDE 설치** → cursor.sh에서 무료 다운로드 (AI 내장 코드 편집기)
3. **GitHub 계정 생성** → github.com
4. **Supabase 계정 생성** → supabase.com (무료)
5. **Vercel 계정 생성** → vercel.com (GitHub으로 로그인)

**프로젝트 시작**:
```
터미널에서:
npx create-next-app@latest my-prompt-builder
cd my-prompt-builder
npm run dev
```
→ 브라우저에서 localhost:3000 열리면 성공

---

### 3단계: UI 개발 (3~5일)

**목표**: 앱의 화면을 만든다

**바이브 코딩 방법**: Cursor에서 AI에게 설명하면 코드 작성

**화면 개발 순서**:

**홈 화면 먼저**:
> "Next.js + Tailwind로 프롬프트 생성 앱의 홈 화면 만들어줘. 상단에 앱 이름, 가운데에 '어떤 프롬프트가 필요하세요?'라는 문구와 카테고리 버튼 6개(마케팅, 글쓰기, 코딩, 번역, 교육, 비즈니스), 하단에 인기 프롬프트 3개 카드 보여줘."

**프롬프트 생성기 화면**:
> "사용자가 목적, 대상, 톤(전문적/친근한/창의적), 출력 형식을 선택하면 프롬프트를 생성해주는 폼 화면 만들어줘. 생성 버튼 클릭 시 결과가 아래에 나타나고 복사 버튼도 있어야 해."

**팁**: 마음에 안 들면 "더 깔끔하게 다시 해줘" "파란색 계열로 바꿔줘" 처럼 대화하듯 수정 요청

---

### 4단계: AI API 연동 (1~2일)

**목표**: 실제로 AI가 프롬프트를 생성하도록 연결

**Claude API 설정**:
1. console.anthropic.com에서 API 키 발급
2. 프로젝트에 환경변수 추가 (.env.local 파일)
3. AI에게 요청: "Claude API를 사용해서 사용자 입력을 받아 최적화된 AI 프롬프트를 생성하는 API 라우트 만들어줘"

**핵심 로직 (AI가 만들어 주는 부분)**:
```
사용자 입력: 목적 + 대상 + 톤 + 형식
→ 내 앱이 Claude API 호출
→ Claude가 최적화된 프롬프트 생성
→ 사용자에게 결과 표시
```

---

### 5단계: 데이터베이스 연동 (1~2일)

**목표**: 템플릿 저장 및 사용자 데이터 관리

**Supabase 설정** (코드 없이 웹에서):
1. Supabase에서 새 프로젝트 생성
2. Table Editor에서 테이블 만들기:
   - `prompts` 테이블: id, title, category, content, created_at
   - `templates` 테이블: id, name, category, description, structure
3. AI에게: "Supabase에서 프롬프트 템플릿을 불러와서 카드로 표시하는 컴포넌트 만들어줘"

---

### 6단계: 배포 (반나절)

**목표**: 인터넷에 공개

**Vercel 배포 순서**:
1. GitHub에 코드 업로드 (Cursor에서 Git 연동)
2. vercel.com → "New Project" → GitHub 저장소 선택
3. 환경변수 입력 (API 키 등)
4. Deploy 버튼 클릭
5. 완료! myapp.vercel.app 주소로 접속 가능

---

### 7단계: 개선 및 운영 (지속)

**데이터 기반 개선**:
- Vercel Analytics (무료)로 어떤 카테고리가 인기있는지 확인
- 사용자 피드백 받기 (간단한 1~5점 평가 기능)
- 매주 새 템플릿 추가

**수익화 방향**:
- 무료: 월 20회 프롬프트 생성
- 프리미엄(월 9,900원): 무제한 + 저장 + 공유
- 기업용: 팀 기능 + API 제공

## 🧠 바이브 코딩 핵심 원칙

### 효과적인 프롬프트 작성법

**나쁜 요청**: "버튼 만들어줘"

**좋은 요청**: "파란색 배경에 흰색 텍스트로 '프롬프트 생성하기'라고 쓰인 둥근 모서리 버튼 만들어줘. 클릭하면 생성 함수가 실행되고 로딩 중에는 스피너가 보여야 해."

### 오류 해결법

오류 메시지가 나오면: "이 오류 해결해줘: [오류 메시지 붙여넣기]"

### 막힐 때

"지금까지 만든 것 요약해줘" → "다음 단계로 뭘 해야 해?" 순서로 질문

## ⚠️ 흔한 실수와 해결책

**실수 1**: 처음부터 완벽하게 만들려 함
→ **해결**: MVP(최소 기능 제품)를 먼저 만들고 개선

**실수 2**: 너무 많은 기능을 한 번에 추가
→ **해결**: 기능 하나씩, 작동 확인 후 다음 기능

**실수 3**: 코드를 이해하려고 너무 많은 시간을 씀
→ **해결**: "이 코드가 무엇을 하는지 한 줄로 설명해줘"로 개념만 파악

**실수 4**: API 키를 GitHub에 올림
→ **해결**: 반드시 .env.local 파일에 저장, .gitignore에 추가
